# 表达式

在 [快速入门](getting-started.md) 和 [控制](controls.md) 中，我们已经使用了一些简单的 **表达式**。表达式是根据其他值来计算值的代码片段。下面我们将进行计算的值和对象主要是 `size_t` 类型

> [!TIP]
> `size_t` 类型是 C 语言中表示**大小**的无符号整数类型，取值范围是 `[0, SIZE_MAX]`。`SIZE_MAX` 的值取决于平台
>
> + 16 位系统：$65,535 = 2^{16} - 1$
> + 32 位系统：$4,294,967,295 = 2^{32} - 1$
> + 64 位系统：$18,446,744,073,709,551,615 = 2^{64} - 1$
>
> `SIZE_MAX` 定义在 `<stdint.h>` 头文件中，可直接使用，无需我们进行计算

在 C 程序中，表达式是由 **操作数** 或 **运算符** 组成的用于计算其他值的代码片段；操作数是运算符作用的对象。C 语言将运算符分为 $3$ 种类型：**操作值** **操作对象** 和 **操作类型**

> [!TIP]
> **值** 是计算的结果，**对象** 是内存中存储值的容器。**赋值 `=` 只能作用于对象，不能作用于值**。

## 值运算符

### 算术运算符

C 语言提了 $5$ 个算术运算符；这些运算符都是 **二元运算符**(它们需要两个操作数)，其工作方式与我们期望的大致相同。

| 运算符 | 形式    | 描述         | 运算符 | 形式    | 描述         |
| ------ | ------- | ------------ | ------ | ------- | ------------ |
| `+`    | `a + b` | 执行加法运算 | `+`    | `+a` | 只提供 `a` 的值 |
| `-`    | `a - b` | 执行减法运算 | `-`    | `-a` | 只提供 `a` 的相反值 |
| `*`    | `a * b` | 执行乘法运算 | `/`    | `a / b` | 执行除法运算 |
| `%`    | `a % b` | 执行模运算   | | | |


```c
size_t a = 45;
size_t b = 7;
size_t c = (a - b) * 2;  // (45 - 7) * 2
size_t d = a - b * 2;    // 45 - 7 * 2
```

这里，`c` 等于 $76$，`d` 等于 $31$。从这个小示例中可以看到，**子表达式可以与括号组合在一起，以实现对运算符的优先绑定**。此外，运算符 `+` 和 `-` 还有一种一元运算符的变体；`-b` 给出 `b` 的负值：值 `a` 可以使 `b + a` 等于 $0$。`+a` 只是提供了 `a` 的值

```c
size_t c = (+a + -b) * 2;
```

即使是对无符号整数，通过运算符 `-` 来求反和差也是 **不错的定义**；换句话说，任意数值输入到减法运算符中，我们总是会计算得到一个有效的结果

这里我们还行需要额外对 `/` 和 `%` 两个运算符多一些解释。在 C 语言种，如果运算符 `/` 的两个操作数都是整数，则执行 **整数除法**，其结果也是整数。例如

```c
int numerator = 11;
int quotient = numerator / 4;
```

从数学上讲，除法运算符 `11 / 4` 的结果当然是 $2.75$。但是，$2.75$ 不是整数，计算机会直接舍弃小数部分的 $0.75$；也就是说，在 C 语言中，`11 / 4` 得到的结果一定是整数 $2$。

> [!TIP]
> 整数除法 `a / b` 表示的是整数 `a` 中有 `k` 个整数 `b`，任何余数 `r` 都会被省略。

运算符 `%` 是对整数除法的补充，它提供了一种在整数除法运算后获取其余数的方式。运算符 `%` 的定义如下：对于任意整数 `x` 和 `y`，我们有 `(x / y) * y + (x % y)` 等于 `x`。这个定义是从 C99 标准开始的

> [!WARNING]
> 显然，运算符 `/` 和 `%` 的右操作数不能为 $0$；否则程序将出现 **未定义行为**

#### 混合算术运算

如果同一个表达式中包含多个运算符时，其含义可能无法一眼就看出。例如表达式 `i + j * k` 的含义可能有两种

1. `i` 加上 `j`，然后再乘以 `k`
2. `j` 乘以 `k`，然后加上 `i`

最简单的方法就是添加括号以实现运算符的优先绑定。例如 `(i + j) * k` 或者 `i + (j * k)`。当然，我们也可以不必这么麻烦的添加圆括号，因为 C 语言按照 **运算符优先级** 来确定那个运算符优先执行。算术运算符的优先级如下表

| 优先级     | 运算符              |
| :--------- | :------------------ |
| 最高优先级 | `+` `-`(一元运算符) |
| 次优先级   | `*` `/` `%`         |
| 最低优先级 | `+` `-`(二元运算符) |

当表达式包含多个相同优先级的运算符时，仅有优先级规则是不够的。这种情况下，**运算符的结合性** 决定了相同优先级的运算符的执行顺序。C 语言提供了 $2$ 种结合性

| 结合性 | 描述                 |
| :----- | :------------------- |
| **左结合** | 运算符是从左向右结合 |
| **右结合** | 运算符是从右向左结合 |

大部分二元运算符(即`*` `/` `%` `+` 和 `-`)都是左结合的；一元运算符(`+` 和 `-`)都是右结合

#### 示例程序: 计算通用产品代码的校验位

UPC-A 共 $12$ 位数字，前 $11$ 位是产品编号，第 $12$ 位是校验位。设 $11$ 位数字为 $d_1 d_2 d_3 \dots d_{11}$，算法步骤：

$$
\text{校验位} = 9 - \Big[\big(3\times(d_1 + d_3 + d_5 + d_7 + d_9 + d_{11}) + (d_2 + d_4 + d_6 + d_8 + d_{10}) - 1\big) \bmod 10\Big]
$$

**注意**：在 UPC-A 的校验位计算中，奇数位权重为 $3$，偶数位权重为 $1$。


```c title="upc.c" linenums="1"
#include <stdio.h>

int main(void) {
    size_t digits[11];

    printf("请输入 11 位数字: ");
    for (size_t i = 0; i < 11; ++i) {
        scanf("%1zu", &digits[i]);
    }

    /* 第一步：奇数位求和 */
    size_t odd_sum = 0;
    for (size_t i = 0; i < 11; i += 2) {
        odd_sum = odd_sum + digits[i];
    }

    /* 第二步：偶数位求和 */
    size_t even_sum = 0;
    for (size_t i = 1; i < 11; i += 2) {
        even_sum = even_sum + digits[i];
    }

    /* 校验位：9 - (odd_sum * 3 + even_sum - 1) % 10 */
    size_t check_digit = 9 - (odd_sum * 3 + even_sum - 1) % 10;

    printf("校验位: %zu\n", check_digit);

    return 0;
}
```

### 比较运算符

在需要bool 类型值的上下文中，我们可以使用 **比较运算符** 来创建一个 `bool` 类型的值。C 语言提供了 $6$ 种比较运算符

| 运算符 | 含义 | 运算符 | 含义     |
| ------ | ---- | ------ | -------- |
| `<`    | 小于 | `<=`   | 小于等于 |
| `>`    | 大于 | `>=`   | 大于等于 |
| `==`   | 等于 | `!=`   | 不等于   |

所有这些运算符都可以用于控制语句，因为比较运算符返回值 `false` 或 `true`。请注意，`false` 或 `true` 都只是 $0$ 或 $1$ 的别名；因此，它们可以用于算术或数组索引

```c
size_t c = (a < b) + (a == b) + (a > b);  // c 的值恒等于 1
size_t d = (a <= b) + (a >= b) - 1;       // 如果 a == b，则 d 的值等于 1；否则等于 0
```

下面的示例代码演示了比较运算符的结果作为数组的索引

```c
double largeA[N] = { /* ... */ };
size_t sign[2] = { 0, 0 };  // sign[false], sign[true]

for (size_t i = 0; i < N; ++i) {
    sign[(largeA[i] < 1.0)] += 1;
}
// sign[0] 统计 >= 1.0 的元素个数
// sign[1] 统计 <  1.0 的元素个数
```

**比较运算符的优先级低于算术运算符**。例如，表达式 `i + j < k - 1` 意思是 `(i + j) < (k - 1)`。关系运算符都是左结合的

> [!WARNING]
> 表达式 `i < j < k` 在 C 语言中是合法的，但是这个表达式的含义不是我们期望的含义。因为 `<` 运算符是左结合的，所以这个表达式等价于 `(i < j) < k`
>
> 换句话说，表达式首先计算 `i < j` 的值，然后用这个子表达式的结果($0$ 或 $1$) 与 `k` 进行比较。这个表达式并不是在测试 `j` 是否位于 `i` 和 `k` 之间；该语义的正确表达式应该是 `i < j && j < k`

### 逻辑运算符

逻辑运算符对已经表示为 `false` 或 `true` 的值进行操作。如果它们不存在，则首先计算操作数的逻辑值(非零值代表 `true`，零值代表 `false`)。C 语言提供了三种基础逻辑运算符

| 逻辑运算符 | 描述                 |
| :--------- | :------------------- |
| `!`        | 逻辑非（一元运算符） |
| `&&`       | 逻辑与               |
| `||`       | 逻辑或               |

下表列出了逻辑运算符的操作结果

| `a`     | `b`     | `!a`    | `a && b` | `a || b` |
| :------ | :------ | :------ | :------- | :------- |
| `true`  | `true`  | `false` | `true`   | `true`   |
| `true`  | `false` | `false` | `false`  | `true`   |
| `false` | `true`  | `true`  | `false`  | `true`   |
| `false` | `false` | `true`  | `false`  | `false`  |

> [!IMPORTANT]
> 从这个表格中，很容易得到下面几个结论
>
> + 逻辑运算符 `!` 就是对逻辑值进行取反: `!false` 的值是 `true`
> + 逻辑运算符 `&&` 要求其两个操作数的值都是 `true` 时，其结果才是 `true`
> + 逻辑运算符 `||` 要求其两个操作数的值都是 `false` 时，其结果才是 `false`

请注意，逻辑运算符的返回值也是 `false` 和 `true`；也可以用作数组的索引

```c
double largeA[N] = { /* ... */ };
size_t isset[2] = { 0, 0 };  // isset[false], isset[true]

for (size_t i = 0; i < N; ++i) {
    isset[!!largeA[i]] += 1;
}
// isset[0] 统计 == 0 的元素个数
// isset[1] 统计 !=  0 的元素个数
```

表达式 `!!largeA[i]` 使用两次逻辑非 `!` 运算符，从而确保 `largeA[i]` 被赋值为 `truth` 值。因此，数组元素 `isset[0]` 和`isset[1]` 将分别保存等于 `0.0` 和不等于 `0.0` 的个数。

逻辑运算符 `&&` 和 `||` 还有一个非常重要的特性 **短路计算**：如果第二个操作数对操作结果不是必需的，那么对它的计算可以省略。对于整数除法，右操作数一定不能为 $0$

```c
// 永远不会出现除以 0 的情况
if (b != 0 && (a / b) > 1) {
    ++x;
}
```

#### 示例程序: 判断贷款风险 

银行发放贷款需要衡量申请人是否有还款能力。下面是判断申请人是否有还款能力的简单标准

+ 年龄大于 $21$ 岁
+ 收入高于 $2.5$ 万或者存款有 $10$ 万

下面的示例程序要求用户提供年龄、收入、存款，并输出用户是否可以申请贷款，贷款额度是多少

```c title="loans.c" linenums="1"
/* loans.c - 判断申请人能否申请贷款 */
#include <stdio.h>

int main(void) {

    int age = {0};      // 年龄
    int income = {0};   // 年收入
    int balance = {0};  // 存款
    int loan = {0};     // 能够申请的额度

    printf("Enter your age in years: ");
    scanf("%d", &age);
    printf("Enter your annual income in dollars: ");
    scanf("%d", &income);
    printf("Enter your current account balance in dollars: ");
    scanf("%d", &balance);

    /* 发放贷款的条件 */
    if (age >= 21 && (income >= 25'000 || balance >= 100'000)) {
        // 能够申请的额度
        // + 2 倍的年收入小于存款的一半；发放 2 倍年收入
        // + 2 倍的年收入大于或等于存款的一半；发放存款的一半
        loan = 2 * income < balance / 2 ? 2 * income : balance / 2;
        printf("You can borrow up to $: %d\n", loan);
    } else {
        printf("Unfortunately, you don't qualify for a loan.\n");
    }
    return 0;
}
```

### 条件运算符

条件运算符类似于 `if` 语句，但它是一个返回所选分支值的表达式。例如，下面的代码片段用于获取两个值中的较大值

```c
int max = i > j ? i : j;
```

这个表达等价于

```c
int max = 0;
if (i > j) {
    max = i;
} else {
    max = j;
}
```

> [!WARNING]
> 很显然，条件表达式使程序更短小但是也带来更大的阅读障碍，因此，我们尽量少的使用它

条件运算符使用 `?` 和 `:` 两个符号组成，它们分隔了条件运算符的三个操作数；也就是说，条件运算符是一个三元运算符

```c
condition? expression1 : expression2;
```

与运算符 `&&` 和 `||` 类似，第二个和第三个操作数仅在真正需要时才计算

> [!TIP]
> 值运算符还有一部分内容没有介绍，就是 **位操作运算符**。这部分内容需要依赖于对象的二进制表示，我们放在 [基本值和数据](basic-types.md) 中介绍

## 对象运算符

其实我们已经使用过了修改对象的运算符，它就是赋值运算符 `=`；这个操作符不是对称的：它右边有一个值，左边有一个对象。有时候，我们也称为 **右值(值)** 和 **左值(对象)**；只要有可能，我们就应该避免使用 **右值** 和 **左值**：谈论 **值** 和 **对象** 就完全足够了

赋值运算符还可以与 **算术运算符** 结合起来的方便缩写。换句话说，有 `+=` `-=` `*=` `/=` 和 `%=` 运算符。例如，在 `for` 循环中，可以使用运算符 `+=`

```c
for (int i = 1, square = 1, odd = 1; i <= number; ++i, odd += 2, square += odd) {
    printf("%10d%10d\n", i, square);
}
```

> [!WARNING]
> 这些运算符必须将所有字符直接连接在一起。例如，用 `i + = 7` 代替 `i += 7` 是语法错误

所有的赋值运算符组成的表达式称为 **赋值表达式**；既然是表达式，那么表达式就应该有一个值，它就是赋值给对象的值。另外还有两个修改对象的运算符：**递增 `++` 运算符** 和 **递减 `--` 运算符**；它们将对象的值增加或减少 $1$

+ `++i` 等价于 `i += 1`
+ `--i` 等价于 `i -= 1`

`++` 和 `--` 既可以放在对象前面(**前缀运算符**)也可以放在对象后面(**后缀运算符**)；它们之间的差别在于表达式求值和修改对象的值这两步操作的顺序。对于前缀运算符，`++i` 会先修改对象的值后对表达式求值；而后缀运算符，`i++` 会先对表达式求值

一般情况下，我们不希望运算符对操作数进行修改。如果运算符对表达式中涉及的对象进行修改，则称运算符具有 **副作用**。在 C 语言中，只有对象运算符才具有副作用。

> [!TIP]
> 一般情况下，值表达式中的副作用是有害的。**不要试图在一条语句中修改多个对象**

## 表达式求值

下表列出了 C 语言中的所有运算符；有些运算符我们会在后续内容逐步介绍完毕。下表中的 **优先级** 数字越大表示的优先级越低。

| 优先级 | 名 称                 | 符 号                                                     | 结合性   |
| :----- | :-------------------- | :-------------------------------------------------------- | :------- |
| 1      | 数组取下标            | `[]`                                                      | 左结合性 |
| 1      | 函数调用              | `()`                                                      | 左结合性 |
| 1      | 取结构和联合的成员    | `.` `->`                                                  | 左结合性 |
| 1      | 自增（后缀）          | `++`                                                      | 左结合性 |
| 1      | 自减（后缀）          | `--`                                                      | 左结合性 |
| 2      | 自增（前缀）          | `++`                                                      | 右结合性 |
| 2      | 自减（前缀）          | `--`                                                      | 右结合性 |
| 2      | 取地址                | `&`                                                       | 右结合性 |
| 2      | 间接寻址              | `*`                                                       | 右结合性 |
| 2      | 一元正号              | `+`                                                       | 右结合性 |
| 2      | 一元负号              | `-`                                                       | 右结合性 |
| 2      | 按位取反              | `~`                                                       | 右结合性 |
| 2      | 逻辑非                | `!`                                                       | 右结合性 |
| 2      | 计算所需空间          | `sizeof`                                                  | 右结合性 |
| 3      | 强制类型转换          | `()`                                                      | 右结合性 |
| 4      | 乘法类运算符          | `*` `/` `%`                                               | 左结合性 |
| 5      | 加法类运算符          | `+` `-`                                                   | 左结合性 |
| 6      | 移位                  | `<<` `>>`                                                 | 左结合性 |
| 7      | 关系                  | `<` `>` `<=` `>=`                                         | 左结合性 |
| 8      | 判等                  | `==` `!=`                                                 | 左结合性 |
| 9      | 按位与                | `&`                                                       | 左结合性 |
| 10     | 按位异或              | `^`                                                       | 左结合性 |
| 11     | 按位或                | `|`                                                      | 左结合性 |
| 12     | 逻辑与                | `&&`                                                      | 左结合性 |
| 13     | 逻辑或                | `||`                                                    | 左结合性 |
| 14     | 条件                  | `?:`                                                      | 右结合性 |
| 15     | 赋值                  | `=` `*=` `/=` `%=` `+=` `-=` `<<=` `>>=` `&=` `^=` `\|=` | 右结合性 |
| 16     | 逗号                  | `,`                                                       | 左结合性 |

这个表用途很广泛，主要的用途之一就是在复杂表达式中确定运算符的执行顺序。例如

```c
a = b += c++ -d + --e / -f;
```

下面我们将根据运算符的优先级并使用圆括号将这个表达式拆分为各个子表达式；方法也非常简单，首先找到最高优先级的运算符，用圆括号将运算符和相应的操作数括起来

```c
// 1. 优先级最高的运算符后缀 ++
a = b += (c++) -d + --e / -f;
// 2. 优先级最高的运算符前缀 -- 和一元 -
a = b += (c++) - d + (--e) / (-f);  // 注意：- d 前面的是减法运算符
// 3. 优先级最高的运算符是 /
a = b += (c++) - d + ((--e) / (-f));
// 4. 优先级最高的运算符是 - 和 +
a = b += (((c++) - d) + ((--e) / (-f)));
// 5. 优先级最高的运算符是 = 和 += 它们是右结合的
a = b += (((c++) - d) + ((--e) / (-f)));
```

有了运算符的优先级和结合性规则，我们就可以将任何 C 语言表达式划分成子表达式。如果表达式是完全括号化的，那么这些规则还可以确定唯一的添加圆括号的方式。与之相矛盾的是，这些规则并不总是允许我们确定表达式的值，表达式的值可能依赖于子表达式的求值顺序

> [!WARNING]
> **C 语言没有定义子表达式的求值顺序**。因此，在表达式 `(a + b) * (c - d)` 中，无法确定子表达式 `(a + b)` 是否在子表达式 `(c - d)` 之前求值

对于没有副作用的表达式，子表达式的求值顺序完全没有任何关系。然而，当表达式有副作用时，表达式的值就可能依赖于子表达式的求值顺序

```c
a = 5;
c = (b = a + 2) - (a = 1);  // 未定义行为
```

> [!WARNING]
> 表达式 `c = (b = a + 2) - (a = 1)` 的值是未定义的。它完全依赖于子表达式 `(a = 1)` 的求值顺序。在编程中，**我们必须避免出现这样的问题**。
